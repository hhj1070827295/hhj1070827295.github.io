# watch与computed之间的区别

computed 属性的结果会被缓存，除非依赖的响应式属性变化才会重新计算。主要当作属性来使用； methods 方法表示一个具体的操作，主要书写业务逻辑； watch 一个对象，键是需要观察的表达式，值是对应回调函数。主要用来监听某些特定数据的变化，从而进行某些具体的业务逻辑操作；可以看作是 computed 和 methods 的结合体；

watch监听的数据，当数据改变时即可调用改方法，可以在该区域内去打印一些东西 computed数据根据实时监听，他可以绑定在数据上

# 闭包相关
    function f1() {
        var n = 123;
        function f2() {    //f2是一个闭包
            alert(n) 
            function f3() { 
                console.log(111)
            }
            return f3()
        }
        return f2();
    }
    f1()
    //这样执行之后得到的结果首先是alert（n）,在打印出111,如果我们把return f3()去掉，那么就不会执行f3()函数，
    //这里包含着一个链式作用域的问题，子对象会一层一层向上级找，但是反过来，父对象无法拿到子对象的值
    闭包概念：能够读取其他函数内部变量的函数。
    或简单理解为定义在一个函数内部的函数，内部函数持有外部函数内变量的引用，
    `用途：  1、读取函数内部的变量
            2、让这些变量的值始终保持在内存中。不会再f1调用后被自动清除。
            3、方便调用上下文的局部变量。利于代码封装。
            原因：f1是f2的父函数，f2被赋给了一个全局变量，f2始终存在内存中，f2的存在依赖f1，因此f1也始终存在内存中，不会在调用结束后，被垃圾回收机制回收。`

原生的setTimeOut是无法使用传参的，但是可以通过闭包实现


    //原生的setTimeout传递的第一个函数不能带参数
    setTimeout(function(param){
        alert(param)
    },1000)


    //通过闭包可以实现传参效果
    function func(param){
        return function(){
            alert(param)
        }
    }
    var f1 = func(1);
    setTimeout(f1,1000);

}
